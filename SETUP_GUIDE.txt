📋 SETUP CHECKLIST
═══════════════════════════════════════════════════════════════════════════════

Follow these steps in order:

☐ 1. CREATE MONGODB ATLAS ACCOUNT
     • Go to: https://www.mongodb.com/cloud/atlas/register
     • Create account and verify email
     • Create a new project
     • Create M0 (free) cluster
     • Create database user and save password
     • Whitelist your IP (or use 0.0.0.0)

☐ 2. GET CONNECTION STRING
     • Click "Connect" in MongoDB Atlas
     • Select "Drivers" connection method
     • Copy the connection string
     • It looks like: mongodb+srv://username:password@cluster.mongodb.net/

☐ 3. CREATE .env FILE
     Create a file named ".env" in your bot root directory:
     
     DISCORD_TOKEN=your_discord_bot_token
     MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/?retryWrites=true&w=majority

     ⚠️  IMPORTANT: Never commit .env to version control!
     ⚠️  Keep this file private - it contains your password!

☐ 4. INSTALL MONGODB DRIVER
     Run in terminal:
     
     npm install

☐ 5. RUN SETUP SCRIPT
     Run in terminal:
     
     node scripts/setupMongoDB.js
     
     This will:
     ✓ Connect to MongoDB
     ✓ Create collections
     ✓ Migrate all data from JSON files
     ✓ Create indexes for performance

☐ 6. START YOUR BOT
     Run in terminal:
     
     node index.js
     
     You should see:
     [1/3] Connecting to MongoDB Atlas...
     ✅ Connected to MongoDB Atlas successfully!
     [2/3] Setting up database indexes...
     ✅ Database indexes created
     [3/3] Syncing data to MongoDB...
     ✅ Bot initialized!

🔄 HOW THE SYSTEM WORKS
═══════════════════════════════════════════════════════════════════════════════

DUAL STORAGE ARCHITECTURE:
  
  Data Update
       ↓
  ┌────────────────────┐
  │   Your Bot Code    │
  └────────────────────┘
       ↓
  ┌────────────────────┐
  │   dataSync Module  │  (Smart Router)
  └────────────────────┘
       ↙              ↘
   ┌──────┐        ┌──────────────┐
   │ JSON │        │   MongoDB    │
   │FILES │        │    ATLAS     │
   └──────┘        └──────────────┘
   Local Files     Cloud Database

• JSON files are the PRIMARY source
• MongoDB Atlas is BACKUP storage
• All saves happen to BOTH simultaneously
• If one fails, the other is used
• On startup, all data is synced to MongoDB

⚠️  IMPORTANT NOTES
═══════════════════════════════════════════════════════════════════════════════

SECURITY:
  • Add .env to .gitignore
  • Never commit .env to version control
  • Keep MONGODB_URI secret (contains password)
  • Keep MongoDB Atlas connection string safe

RELIABILITY:
  • Data automatically backs up to MongoDB
  • If MongoDB is down, bot still works (uses files)
  • If files are corrupted, MongoDB has clean copy
  • Run setupMongoDB.js to resync if needed

PERFORMANCE:
  • Uses caching to avoid constant reloads
  • Batches multiple saves together
  • Creates indexes for fast queries
  • Minimal performance impact

🆘 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Problem: "MONGODB_URI not set"
Solution: Create .env file with MONGODB_URI

Problem: "Failed to connect to MongoDB"
Solution: 
  1. Check MongoDB Atlas cluster is running
  2. Verify connection string is correct
  3. Check IP whitelist in MongoDB security
  4. Verify username/password

Problem: "npm install mongodb failed"
Solution:
  1. Delete node_modules and package-lock.json
  2. Run npm install again

Problem: Data not syncing
Solution:
  1. Check bot console for error messages
  2. Run: node scripts/setupMongoDB.js
  3. Restart bot

For complete help: See MONGODB_SETUP.md
