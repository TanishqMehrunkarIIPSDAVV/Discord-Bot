📊 DATA FLOW ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════════

WRITE FLOW (Saving Data):
    Bot Code
       ↓
    saveData() or saveDocument()
       ↓
    ┌─────────┬─────────┐
    ↓         ↓
   JSON     MongoDB
   File     Atlas
    ↓         ↓
  Saved    Synced

READ FLOW (Loading Data):
    Bot Code
       ↓
    loadData()
       ↓
    Try File
       ↓ (if file not found)
    Try MongoDB
       ↓ (if both fail)
    Return Default
       ↓
    Return Data

🚀 GETTING STARTED - 3 STEPS
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Setup MongoDB Atlas
   • Go to: https://www.mongodb.com/cloud/atlas/register
   • Create account → Create cluster → Get connection string
   • Time: ~5 minutes

STEP 2: Configure Bot
   • Create .env file with MONGODB_URI
   • Run: npm install
   • Run: node scripts/setupMongoDB.js
   • Time: ~2 minutes

STEP 3: Start Bot
   • Run: node index.js
   • Bot will sync data to MongoDB automatically
   • Check console for confirmation
   • Time: ~1 minute

TOTAL TIME: ~8 minutes to full setup! ⏱️

🔧 HOW TO USE IN YOUR CODE
═══════════════════════════════════════════════════════════════════════════════

BEFORE (Old way - file only):
    const fs = require("fs");
    const path = require("path");
    
    const data = JSON.parse(fs.readFileSync(filePath, "utf8"));
    // ... modify data ...
    fs.writeFileSync(filePath, JSON.stringify(data));

AFTER (New way - dual storage):
    const { loadData, saveData } = require("./utils/dataSync");
    
    const data = await loadData("quest-state", {});
    // ... modify data ...
    await saveData("quest-state", data);

Benefits:
✓ Automatically saves to files AND MongoDB
✓ Falls back if one system fails
✓ Much simpler code
✓ Cloud backup included
✓ No extra work needed
